about How RepoPilot reads a repo

RepoPilot pulls live data from the GitHub API — README, languages, contributors, issues, and comment activity — then layers an AI pass on top to translate raw signals into plain-language guidance: what the project does, which folders matter, where a first-time contributor should start, and when maintainers are most likely to respond.

Health metrics (bus factor, issue closing time, responsiveness, staleness) are computed directly from repository activity, so the score reflects how the project is actually run — not just how it's described.
